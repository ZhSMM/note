# Shell

##### 书写格式

linux推荐使用`.sh`作为文件的扩展名，脚本文件书写格式如下：

1. 脚本文件第一行要求使用 shebang 即`#!`指定一个脚本的解释器，如`/usr/bin/python`、`#!/usr/bin/bash`

2. 脚本文件注释：

   - #：单行注释
   - <<：多行注释，<<后可以跟任意字符串，但前面使用什么关键词，结束注释时必须使用相同的关键词

   ```sh
   # 单行注释
   
   :<<comment
   多行注释
   @description 脚本功能描述
   @author 作者
   @date 2022/09/26
   comment
   ```



##### 脚本执行


脚本文件文件执行：

```sh
# 脚本文件没有可执行权限：使用sh或指定shell执行
sh demo.sh
bash demo.sh

# 脚本具备可执行权限：使用相对路径或绝对路径执行
chmod +x demo.sh
./demo.sh

# 开启子进程执行
pstree                      # 查看进程间的关系
sh demo.sh 或 ./demo.sh     # 执行时均会开启子进程
source demo.sh 或 . demo.sh # 执行时在当前进程执行
```

验证是否开启子进程：

```sh
# 编写exit.sh
#!/bin/sh
exit

# 使用两种方式分别执行
sh exit.sh     # 不影响当前shell
source exit.sh # 会直接关闭当前shell
```



##### 数据输出

echo：用于显示字符串信息，支持如下参数

- -n：echo默认输出的最后会添加换行符，-n参数会删除默认的换行符

- -e：使echo命令可以识别字符串中的转义符号

  常见的转义符号：

  | 符号     | 功能                                       |
  | -------- | ------------------------------------------ |
  | \b       | backspace，退格键                          |
  | \f       | 换行但光标仍停留在原来的位置               |
  | \n       | 换行但光标移至行首                         |
  | \r       | 光标移至行首，但不换行                     |
  | \t       | 插入Tab键                                  |
  | \\       | 打印\                                      |
  | \033或\e | 设置终端属性，如字体颜色、背景颜色、定位等 |

  示例：

  ```sh
  # \e[0m：关闭属性设置
  # \e[1m或\033[1m：字体加粗
  echo -e "\e[1mok\e[0m"
  
  # \e[4m：加下划线
  echo -e "\e[4mok\e[0m"
  
  # \e[5m：闪烁显示
  echo -e "\e[5mok\e[0m"
  
  # 设置字体颜色
  # 字体颜色：30-黑色、31-红色、32-绿色、33-棕色、34-蓝色、35-紫色、36-蓝绿色、37-亮灰色
  # 背景颜色：40-黑色、41-红色、42-绿色、43-棕色、44-蓝色、45-紫色、46-蓝绿色、47-亮灰色
  # 输出红色字体，绿色背景的ok
  echo -e "\e[31;42mok\e[0m"
  
  # 使用H定义位置属性
  # 在屏幕第3行，第4列输出ok
  echo -e "\e[3;4HOK"
  ```



printf：格式化输出数据（默认不换行），`printf [格式] 参数`，常用的格式化字符串如下

| 格式字符 | 功能               | 格式字符 | 功能                       |
| -------- | ------------------ | -------- | -------------------------- |
| %d或%i   | 十进制整数         | %o       | 八进制整数                 |
| %x       | 十六进制整数       | %u       | 无符号十进制整数           |
| %f       | 浮点数（小数点数） | %s       | 字符串                     |
| \b       | backspace，退格键  | \f       | 换行但光标仍停留在原来位置 |
| \n       | 换行且光标移至行首 | \r       | 光标移至行首，但不换行     |
| \t       | Tab键              | %u       | 超大整数值                 |

示例：

```sh
# 左对齐
printf "%-5d%-5d%-5d" 12 14 16

# 右对齐
printf "%5d%5d%5d" 12 14 16

# 小数点后保留3位
printf "%.3f" 12.5
```

| 选项 | 功能                                                    |
| ---- | ------------------------------------------------------- |
| -p   | 显示提示信息                                            |
| -t   | 设置读入数据的超时时间                                  |
| -n   | 设置读取n个字符后结束（默认会读取标准输入的一整行内容） |
| -r   | 支持读取\（默认read命令认为\为转义字符）                |
| -s   | 静默模式，不显示标准输入的内容                          |

##### 重定向

在linux中，标准输入的文件描述符为0，标准输出的文件描述符为1，标准错误输出的文件描述符为2。

```sh
# > 等价于 1>：覆盖重定向标准输出（会丢失原有数据，文件不存在时会创建）
# >> 等价于 1>>：追加重定向标准输出
# 2>、2>>：覆盖、追加重定向标准错误流
# &>、&>>：同时将标准输出和错误输出都进行重定向
# 2>&1、2>>&1：将错误输出重定向到标准正确输出
# 1>&2、1>>&2：将标准正确输出重定向到错误输出
echo "1" > demo.txt

# /dev/null：特殊设备，可以吞掉写入的数据
echo "hello" &> /dev/null

# <<：将数据内容重定向传递给前面的一个命令，作为命令的输入
# 命令 << 分隔符
# 内容
# 分隔符
cat > /tmp/test.txt << -EOF
写入测试文件的内容
EOF
```

示例：自动分区脚本

```sh
#!/bin/sh
# 自动将vdb整个磁盘分成一个区，并将该分区格式化（注意所有数据都将丢失）
# n（新建分区），p（新建主分区），1（主分区编号为1）
# 回车键，空白行（从磁盘哪个位置开始分区，默认从第1个扇区）
# 回车键，空白行（分区到哪个扇区结束，回车键代表最后，将整个磁盘分1个区）
# wq（保存退出），mkfs.xfs（格式化分区）
# <<-：忽略所有数据内容和分隔符之间的Tab键
fdisk /dev/vdb << -EOF
n
p
l


wq
EOF

mkfs.xfs /dev/vdb1
```



##### 变量

变量使用：

```sh
# 变量定义：变量名=变量值
KEY=13

# 定义环境变量：export 变量名（将自定义变量转换成环境变量）
export var_name

# 变量引用：$变量名、${变量名}
# 查看变量：echo $变量名、set（显示所有变量，自定义和环境变量）
echo $var_name

# 查看环境变量
env | grep var_name

# 取消变量：unset 变量名（仅在当前shell生效）
unset var_name
```

示例：获取主机数据（网卡IP、可用内存、CPU负载）

```sh

```

##### sed

sed：Stream Editor，流编辑器，对标准输出或文件逐行进行处理。其会逐行扫描输入的数据，并将读取的数据内容复制到缓冲区中，称为模式空间，然后拿模式空间中的数据与给定的条件进行匹配，如果匹配成功则执行特定的sed指令，否则sed会跳过输入的数据行，继续读取后续的数据。默认情况下sed会把最终的数据结果通过标准输出显示在屏幕上。

- 语法格式：

  ```sh
  # 通过管道从标准输出流获取数据
  stdout | sed [option] "/pattern/command"
  
  # 从文件中获取数据
  sed [option] "/pattern/command" file
  ```

- 选项：

  | 选项         | 含义                                                      |
  | ------------ | --------------------------------------------------------- |
  | -n，--silent | 屏蔽默认输出功能（默认sed会将匹配到的数据显示在屏幕上）   |
  | -e           | 指定需要执行的sed指令，支持使用多个-e参数                 |
  | -f           | 编辑动作保存在文件中，指定文件执行                        |
  | -r           | 支持扩展正则表达式                                        |
  | -i[SUFFIX]   | 直接修改文件内容，如果设置了SUFFIX后缀名，sed会将数据备份 |

- pattern模式：

  ```sh
  # $：匹配最后一行
  sed -n '$' /etc/passwd
  
  # !：对匹配条件进行取反操作
  # 匹配除最后一行的所有行
  sed -n '$!' /etc/passwd
  
  # first~step：从first行开始，步长为step，匹配所有满足条件的数据行
  10~2
  
  # 匹配第10行
  10command
  
  # 匹配从第10行开始，到第20行结束
  10,20command
  
  # 匹配从第10行开始，到第16行结束
  10,+5command
  
  # 匹配到pattern1的行
  /patter1/command
  
  # 匹配到pattern1的行开始，到匹配pattern2的行结束
  /patter1/,/pattern2/command
  
  # 匹配从第10行开始，到匹配到pattern1的行结束
  10,/pattern1/command
  
  # 匹配到pattern1的行开始，到第10行匹配结束
  /pattern1/,10command
  ```

##### awk

awk：文本处理工具，通常用于处理数据并生成结果报告

- 语法格式：

  ```sh
  # 从文件中读取数据
  awk 'BEGIN{}pattern{commands}END{}' file_name
  
  # 从标准输出流获取数据
  standard output | awk 'BEGIN{}pattern{commands}END{}'
  ```

  格式说明：

  | 语法格式   | 说明                     |
  | ---------- | ------------------------ |
  | BEGIN{}    | 正式处理数据之前执行     |
  | pattern    | 匹配模式                 |
  | {commands} | 处理命令，可能多行       |
  | END{}      | 处理完所有匹配数据后执行 |

- 模式匹配的两种用法：

  - RegExp：

    | 比较符号 | 描述                                 |
    | -------- | ------------------------------------ |
    | //       | 全行数据正则匹配                     |
    | !//      | 对全行数据正则匹配后取反             |
    | ~//      | 对特定数据正则匹配                   |
    | !~//     | 对特定数据正则匹配                   |
    | ==       | 等于                                 |
    | !=       | 不等于                               |
    | >        | 大于                                 |
    | >=       | 大于等于                             |
    | <        | 小于                                 |
    | <=       | 小于等于                             |
    | &&       | 逻辑与，与A&&B，要求满足A并且满足B   |
    | \|\|     | 逻辑或，如A\|\|B，要求满足A或者满足B |

    ```sh
    # 匹配含有root字符串的行
    awk 'BEGIN{FS=":"}/root/{print $0}' /etc/passwd
    
    # 匹配/etc/passwd中以yarn开头的所有行
    awk 'BEGIN{FS=":"}/^yarn/{print $0}' /etc/passwd
    ```

  - 关系运算匹配

    | 符号 | 说明           | 符号 | 说明             |
    | ---- | -------------- | ---- | ---------------- |
    | <    | 小于           | >    | 大于             |
    | <=   | 小于等于       | >=   | 大于等于         |
    | ==   | 等于           | !=   | 不等于           |
    | ~    | 匹配正则表达式 | !~   | 不匹配正则表达式 |

    布尔运算符匹配：`||`（或）、`&&`（与）、`!`（非）

    示例：

    ```sh
    # 匹配第三个字段小于50的所有行
    awk 'BEGIN{FS=":"}$3<50{print $0}' /etc/passwd
    
    # 匹配第三个字段为/bin/bash的所有行
    awk 'BEGIN{FS=":"}$3=="/bin/bash"{print $0}' /etc/passwd
    
    # 匹配第三个字段包含3个以上数字的所有行信息
    awk 'BEGIN{FS=":"}$3~/[0-9]{3,}/{print $0}' /etc/passwd
    ```

- 内置变量：

  | 内置变量 | 含义                                            |
  | -------- | ----------------------------------------------- |
  | $0       | 整行内容                                        |
  | $1-$n    | 当前行的第1到n个字段                            |
  | NF       | 当前行的字段个数，即多少列                      |
  | NR       | 当前行的行号，从1开始计数                       |
  | FNR      | 多文件处理时，每个文件行号单独计数，都是从1开始 |
  | FS       | 输入字段分隔符，不指定默认以空格或Tab键分割     |
  | RS       | 输入行分隔符，默认回车换行                      |
  | OFS      | 输出字段分隔符，默认为空格                      |
  | ORS      | 输出行分隔符，默认为回车换行                    |
  | FILENAME | 当前输入的文件名称                              |
  | ARGC     | 命令行参数个数                                  |

- awk支持关联数组：数组的索引下标可以是任意字符或数字（可以不连续），使用数字作为索引时awk会自动将数字转换成字符，如果使用字符作为索引则需要使用引号括起来

  ``` sh
  # 一维数组：数组名[索引]=值
  # 多维数组：数组名[索引1][索引2]=值、数组名[索引1, 索引2]=值
  # 取值：数组名[索引]
  awk 'BEGIN{tom["age"]=22; tom["height"]=178; \
  print tom["age"], tom["height"]}'
  
  # 遍历数组：for (索引 in 数组名) { 动作指令序列 }
  # 通过循环的方式输出数组索引或者元素值时并不会按照输入的顺序输出，是无序的
  # 通过循环获取数组索引
  awk 'BEGIN{tom["age"]=22; tom["height"]=178; \
  for (i in tom) { print i }}'
  # 通过循环获取数组元素的值
  awk 'BEGIN{tom["age"]=22; tom["height"]=178; \
  for (i in tom) { print tom[i] }}'
  
  # 判断某个索引是否为数组成员：if(索引 in 数组){}
  awk 'BEGIN{tom["age"]=22; tom["height"]=178; \
  if ('score' in tom) { print "YES" } else { print "NO" }}'
  
  # 统计主机上所有TCP连接状态数，按照每个TCP状态分类
  netstat -an | grep tcp | awk '{array[$6]++}END{for(a in array) print a,  array[a]}'
  
  # 示例：计算横向数据综合，计算纵向数据总和
  # 测试数据：score.txt
  allen 80 90 87 91
  mike  78 86 93 96
  Kobe  66 92 82 78
  Jerry 98 74 66 54
  Wang  87 21 99 43
  # 统计命令
  awk 'BEGIN{printf "%-10s%-10s%-10s%-10s%-10s%-10s\n","Name","Yuwen","Math","English","Physical","total"}{total=$2+$3+$4+$5;yuwen+=$2;math+=$3;english+=$4;physical+=$5;sum+=total;printf "%-10s%-10s%-10s%-10s%-10s%-10s\n",$1,$2,$3,$4,$5,total}END{printf "%-10s%-10s%-10s%-10s%-10s%-10s\n","",yuwen,math,english,physical,sum}' score.txt
  ```

  | 函数                        | 用途                                                         |
  | --------------------------- | ------------------------------------------------------------ |
  | getline                     | 读取下一条记录并复制给$0，并重新设置NF、NR和FNR，会继续执行剩下的指令 |
  | next                        | 停止处理当前的输入记录，立刻读取下一条记录并返回awk程序的第一个模式匹配重新处理数据 |
  | system                      | 启动一个新的shell执行相关命令                                |
  | cos(expr)                   | 返回expr的cosine值                                           |
  | sin(expr)                   | 返回expr的sine值                                             |
  | sqrt(expr)                  | 返回expr的平方根                                             |

  | rand()                      | 返回0到1之间的随机数N（0＜=N＜1）                            |
  | srand([expr])               | 使用expr定义新的随机数种子，没有expr时则使用当前系统的时间为随机数种子 |
  | length([s])                 | 统计字符串s的长度，如果不指定字符串s则统计$0的长度           |
  | index(字符串, 字符串2)      | 返回字符串2在字符串1中的位置坐标                             |

  | sub(r, s, [, t])            | 与gsub类似，但仅替换第一个匹配的字符串，而不是替换全部       |
